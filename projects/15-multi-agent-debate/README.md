# 15-multi-agent-debate：多 Agent 辩论与投票决策

> 星际殖民委员会 — 三位专家围绕殖民地选址展开结构化辩论

## 项目概述

实现多个独立 Agent 角色围绕殖民地选址进行结构化辩论（带工具查证），通过三阶段论证 + 投票机制达成决策。

### 核心学习目标

- **多 Agent 协作**：独立推理 → 交叉质疑 → 收敛共识
- **信息隔离与注入**：第一轮各自独立，第二轮注入所有人发言
- **工具查证辩论**：论点用数据支撑，非纯推理
- **投票机制**：结构化投票 + 立场变化追踪

## 场景：星际殖民委员会

三颗候选星球：

| 星球 | 特点 | 宜居 | 防御 | 经济 |
|------|------|------|------|------|
| 🔵 蓝晶星 | 海洋行星，含水量85% | 87 | 72 | 78 |
| 🔴 赤焰星 | 火山行星，稀有资源丰富 | 15 | 65 | 85 |
| 🟢 翡翠星 | 丛林行星，生物多样性极高 | 82 | 55 | 70 |

三位委员会专家：

| Agent | 角色 | 关注点 | 专属工具 |
|-------|------|--------|----------|
| 🔬 科学官 | 首席科学顾问 | 宜居性、生态、科研 | `analyze_habitability()` |
| ⚔️ 军事官 | 安全顾问 | 防御、战略、威胁 | `assess_defense()` |
| 💰 经济官 | 资源顾问 | 资源、成本、回报 | `evaluate_economics()` |

## 辩论三阶段

```
第 1 阶段：独立立论
├── 各 Agent 独立查数据
├── 不看其他人观点（信息隔离）
└── 输出推荐 + 理由

第 2 阶段：交叉质疑
├── 看到所有人第一轮发言
├── 针对性反驳或补充
└── 可用工具查数据反驳

第 3 阶段：总结投票
├── 看到所有历史发言
├── 最终陈述
└── 结构化投票（JSON）
```

## 工具设计：共享 + 专属

**共享工具**（所有 Agent 可用）：
- `search_planets(query)` — 搜索候选星球
- `lookup_planet(planet, field)` — 精确查询属性

**专属工具**（每角色独有）：
- 科学官：`analyze_habitability(planet)` — 综合宜居性分析
- 军事官：`assess_defense(planet)` — 综合防御态势评估
- 经济官：`evaluate_economics(planet)` — 综合经济价值评估

## 关键设计决策

1. **信息隔离**：第一轮每个 Agent 独立调用 `run_agent()`，只看知识库不看其他人
2. **信息注入**：第二轮通过 `PHASE_PROMPTS["交叉质疑"]` 模板注入所有人第一轮发言
3. **工具查证**：Agent 在辩论中可以随时用工具查数据，论点有数据支撑
4. **投票解析**：优先 JSON 解析，兜底启发式提取（容错设计）
5. **立场追踪**：对比第一轮推荐 vs 第三轮投票，展示立场变化
