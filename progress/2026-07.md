### 学到的内容
- **RAG 在记忆系统中的应用**：向量检索 + 注入 context = 让 Agent "记住"过去的对话，本质是 Retrieval-Augmented Generation
- **ChromaDB 的 cosine distance**：返回的是距离（越小越相似），`similarity = 1 - distance` 才是相似度
- **自定义 EmbeddingFunction**：继承 `chromadb.EmbeddingFunction`，`__call__` 接收 `list[str]` 返回 `list[list[float]]`，完全通过 API，无本地模型
- **PersistentClient vs EphemeralClient**：PersistentClient 写磁盘，重启后数据保留；EphemeralClient 仅内存，程序退出即丢失
- **new_session 的关键设计**：只清 `_short_term`（当前 session messages），不动 ChromaDB，这才是"跨 session 记忆持久化"的本质
- **相似度阈值的工程意义**：无关历史记忆注入 context 会干扰 LLM 推理，threshold=0.7 是过滤噪声的实用值

### 遇到的问题
- **ChromaDB 默认 embedding 会下载本地模型**：不传 `embedding_function` 时 ChromaDB 自动用 `sentence-transformers/all-MiniLM-L6-v2`，必须显式传入自定义 EF
- **Collection 空时 query 崩溃**：`collection.query()` 在 `count()==0` 时抛异常，需要提前判断返回 `[]`

- ✅ 通过全部验收：json_schema 100% 校验通过、refusal 处理、JSON 语法错误处理

### 学到的内容
- **OpenAI Structured Outputs strict 模式**：受约束解码保证输出严格匹配 schema，但要求 `additionalProperties: false` + 所有属性进 `required`
- **Pydantic v2 的 $defs 机制**：嵌套 BaseModel 定义放在 `$defs`，主 schema 通过 `$ref` 引用，strict 处理必须递归到 $defs
- **三种模式的可靠性梯度**：json_schema（0 重试，100%）> json_object（需校验+重试）> text（最不可靠，需从自由文本提取 JSON）
- **重试反馈的关键**：把 ValidationError 提取成可读错误信息追加到 messages，让 LLM 定向修正，而非无脑重试
- **refusal 字段**：json_schema 模式下模型拒绝时填充 `message.refusal` 而非 `content`，需防崩溃处理

### 遇到的问题
- **Pydantic 直接生成的 schema 无法过 strict 校验**：需 `_enforce_strict_schema()` 递归补 `additionalProperties: false` 和 `required`
- **text 模式 JSON 提取格式多样**：纯 JSON / markdown 代码块 / 解释文本+JSON，需多策略解析

### 学到的内容
- **Plan→Execute→Re-plan 双层认知架构**：外层规划 DAG，内层 ReAct 执行单任务，区别于 03（单步推理）和 04（重试修正）
- **压平的 DAG 优于嵌套树**：用 `depends_on` 字段表达跨分支依赖，嵌套 JSON 无法表达；靠拓扑排序 + depth 计算还原树形展示
- **Kahn 拓扑排序天然带环检测**：排序节点数 < 总节点数即存在环，无需额外算法
- **局部重规划的核心是依赖重定向**：失败子任务拆成子树后，原来依赖它的下游任务要自动重定向到新子树的出口节点
- **用结构化收尾工具判定成败**：report_result(success, reason) 比解析自然语言可靠，reason 直接作为重规划的失败输入
- **把"部分完成状态"编码给 LLM**：重规划时用 `to_display_rows()` + `get_progress()` 把当前树状态文本化，让 LLM 只重规划受影响子树

### 遇到的问题
- **动态展开导致 parent 状态回退**：子任务展开出新子节点后，父节点状态管理需谨慎，避免已完成状态被覆盖
- **ANSI 着色破坏 ASCII 树对齐**：颜色码计入字符串宽度导致列错乱，需固定宽度或手动对齐
- **无限循环风险**：需 max_replans + 无 ready 任务检测 + 总迭代上限三重兜底
- **executor verbose 与 display 职责重叠**：主循环硬编码 `verbose=False`，执行渲染统一交给 display，避免双打印（executor verbose 保留供模块独立自测）
